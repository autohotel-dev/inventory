## 2024-05-18 - [Optimizing Frequent Array Methods in React]
**Learning:** React components containing both search terms (`onChange` state variables triggered every keystroke) and large array renders (like sorting and filtering list items) often re-run `.sort()` and `.filter()` operations on every state update, which can block the main thread and feel sluggish on large data sets (e.g., employee lists). Adding `useMemo` avoids redundant sorts and filters unless the core data or specific filter criteria actually change.
**Action:** When identifying performance bottlenecks where large arrays are filtered/sorted *and* local state changes rapidly (like text input fields), prioritize wrapping these array transformations in `useMemo`.
## 2025-02-12 - [Performance Optimization: Single-pass iteration and Memoization]
**Learning:** Found an anti-pattern in React components rendering lists (e.g. `advanced-kardex-view.tsx` and `advanced-stock-view.tsx`) where multiple `.filter().length` calls were used to calculate specific statuses (e.g., IN, OUT, ADJUSTMENTS) during the render cycle. This results in multiple unnecessary traversals of potentially large arrays of data (O(N * M) instead of O(N)).
**Action:** Always consolidate multiple map/filter iterations into a single `.reduce()` pass returning an accumulator object with the desired metrics. Furthermore, always wrap the resulting derived states using `useMemo` so the work is not redone needlessly on each React re-render. Make sure to apply this to similar metric calculations across the application.
