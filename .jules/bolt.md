## 2024-05-18 - [Optimizing Frequent Array Methods in React]
**Learning:** React components containing both search terms (`onChange` state variables triggered every keystroke) and large array renders (like sorting and filtering list items) often re-run `.sort()` and `.filter()` operations on every state update, which can block the main thread and feel sluggish on large data sets (e.g., employee lists). Adding `useMemo` avoids redundant sorts and filters unless the core data or specific filter criteria actually change.
**Action:** When identifying performance bottlenecks where large arrays are filtered/sorted *and* local state changes rapidly (like text input fields), prioritize wrapping these array transformations in `useMemo`.
## 2024-05-18 - [Optimizing Array Count Chains]
**Learning:** React components that calculate multiple derived metrics from a single array often use chaining, such as `array.filter(c => c.status === 'X').length`. Running this multiple times for different conditions (`X`, `Y`, `Z`) loops over the array entirely each time. This creates an O(N * M) penalty where N is array length and M is the number of distinct statistics tracked. Moving these inside a single `useMemo` block using a single `array.reduce()` pass computes all stats in O(N) time and drastically reduces CPU overhead during fast state updates (like typing in a search bar).
**Action:** Replace sequential `.filter(...).length` statements with a single `.reduce()` whenever more than two statistics are being derived from a changing array in a React component's render body.
