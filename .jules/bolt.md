## 2024-05-18 - [Optimization of multiple map/filter passes]
**Learning:** Found a significant performance bottleneck where a component was performing 4 separate `O(n)` array scans (`.filter()` + `.find()`) inside its main render body on every re-render. Since React component state triggers frequent renders, these $O(N)$ operations compounded quickly.
**Action:** When finding multiple `.filter()`, `.reduce()`, or `.sort()` operations on the same dataset, consolidate them into a single `useMemo` block that uses a single iteration pass (`.forEach` or `for` loop) to derive all needed sub-arrays. Crucially, pay attention to the order of operations when sorting: if one array slice relies on the sorted result of another slice, ensure it is derived *after* the sorting occurs, not during the initial unsorted pass. Also, always place hooks before any early return statements.
